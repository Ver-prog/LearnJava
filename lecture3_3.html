Лекція 3.3: ООП і моделювання: наслідування і поліморфізм

Наслідування та поліморфізм — це основні принципи об'єктно-орієнтованого програмування, які дозволяють створювати більш ефективні, зручні для підтримки та розширення програми. Вони дозволяють зменшити дублювання коду, полегшити моделювання реальних об'єктів, а також забезпечити більшу гнучкість у програмі.

1.1. Наслідування

Наслідування є механізмом, за допомогою якого один клас може успадковувати властивості та методи іншого класу. Це дозволяє створювати нові класи, засновані на вже існуючих, не пишучи повторно один і той самий код.

У мові Java наслідування реалізується через ключове слово extends.


class БатьківськийКлас {
    String ім'я;

    public БатьківськийКлас(String ім'я) {
        this.ім'я = ім'я;
    }

    public void привітання() {
        System.out.println("Привіт, я " + ім'я);
    }
}

class Підклас extends БатьківськийКлас {
    int вік;

    public Підклас(String ім'я, int вік) {
        super(ім'я);
        this.вік = вік;
    }

    @Override
    public void привітання() {
        System.out.println("Привіт, я " + ім'я + " і мені " + вік + " років");
    }
}

public class Main {
    public static void main(String[] args) {
        Підклас особа = new Підклас("Іван", 25);
        особа.привітання();  // Виведе: Привіт, я Іван і мені 25 років
    }
}

1.2. Переваги наслідування:

  • Повторне використання коду: підкласи успадковують властивості та методи батьківських класів, що дозволяє уникати дублювання.
  • Ієрархія класів: створення чіткої ієрархії між класами та об'єктами.
  • Модифікація та доповнення функціоналу: підклас може додавати нові методи або перевизначати існуючі методи батьківського класу.

2. Поліморфізм

Поліморфізм дозволяє використовувати однакові методи для об'єктів різних класів. Це забезпечує гнучкість і дозволяє обробляти різні типи об'єктів через один інтерфейс. У Java поліморфізм реалізується завдяки наслідуванню та перевизначенню методів.

2.1. Поліморфізм через перевизначення методів

Поліморфізм у Java часто досягається через перевизначення методів в підкласах. Це означає, що підклас може мати власну реалізацію методу, яка замінює реалізацію з батьківського класу.


class Тварина {
    public void звук() {
        System.out.println("Тварина робить звук");
    }
}

class Собака extends Тварина {
    @Override
    public void звук() {
        System.out.println("Гав!");
    }
}

class Кіт extends Тварина {
    @Override
    public void звук() {
        System.out.println("Мяу!");
    }
}

public class Main {
    public static void main(String[] args) {
        Тварина тварина1 = new Собака();
        Тварина тварина2 = new Кіт();

        тварина1.звук();  // Виведе: Гав!
        тварина2.звук();  // Виведе: Мяу!
    }
}

2.2. Поліморфізм через інтерфейси

Інтерфейси також є важливою частиною поліморфізму в Java. Клас може реалізовувати кілька інтерфейсів, що дозволяє створювати загальні інтерфейси для різних класів.


interface Звук {
    void видаватиЗвук();
}

class Собака implements Звук {
    @Override
    public void видаватиЗвук() {
        System.out.println("Гав!");
    }
}

class Кіт implements Звук {
    @Override
    public void видаватиЗвук() {
        System.out.println("Мяу!");
    }
}

public class Main {
    public static void main(String[] args) {
        Звук собака = new Собака();
        Звук кіт = new Кіт();

        собака.видаватиЗвук();  // Виведе: Гав!
        кіт.видаватиЗвук();     // Виведе: Мяу!
    }
}

2.3. Переваги поліморфізму:

  • Гнучкість: можна використовувати один інтерфейс для обробки об'єктів різних типів.
  • Зниження залежностей: код стає менш залежним від конкретних реалізацій, що полегшує модифікації.
  • Масштабованість: дозволяє додавати нові класи без зміни існуючого коду, що значно полегшує підтримку програми.

3. Зв'язок між наслідуванням і поліморфізмом

Наслідування та поліморфізм тісно пов'язані. Наслідування дозволяє створювати нові класи на основі вже існуючих, що забезпечує можливість перевизначення методів. Поліморфізм дозволяє працювати з об'єктами різних класів через спільний інтерфейс, забезпечуючи гнучкість у програмі.


class Main {
    public static void main(String[] args) {
        Тварина собака = new Собака();
        Тварина кіт = new Кіт();
        
        викликатиЗвук(собака);  // Виведе: Гав!
        викликатиЗвук(кіт);     // Виведе: Мяу!
    }

    public static void викликатиЗвук(Тварина тварина) {
        тварина.звук();
    }
}

4. Висновок

Наслідування та поліморфізм є важливими інструментами для створення ефективних, масштабованих та зручних для підтримки програм. Вони дозволяють значно знизити кількість повторюваного коду, покращити організацію ієрархії класів і забезпечити більшу гнучкість при роботі з об'єктами різних типів. Мова Java надає потужні інструменти для реалізації цих принципів, що робить її зручною для розробки великих і складних програм.
