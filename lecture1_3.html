Лекція 1.3: Основи програмування: умовні конструкції

Умовні конструкції є одними з основних елементів програмування, які дозволяють програмам виконувати різні дії залежно від умов. Вони дозволяють приймати рішення на основі вхідних даних або поточного стану програми. Умовні конструкції широко використовуються в програмуванні для реалізації логіки, яка змінюється в залежності від результатів порівнянь, значень змінних чи введених даних.

У цій лекції ми розглянемо три основні види умовних конструкцій: if, else та switch. Ми розглянемо їхню структуру, принцип роботи, а також приклади використання в різних ситуаціях.

1. Умовний оператор if

Оператор if використовується для виконання певного блоку коду, якщо задана умова є істинною (true). У випадку, якщо умова є хибною (false), цей блок не виконується.

if (умова) {
    // код, який виконується, якщо умова істинна
}

Приклад:

int a = 10;
if (a > 5) {
    System.out.println("a більше за 5");
}

2. Умовний оператор if-else

Оператор if-else дозволяє виконати один блок коду, якщо умова істинна, і інший блок, якщо умова хибна. Це дає змогу зробити програму більш гнучкою, виконуючи різні дії в залежності від результату перевірки умови.

if (умова) {
    // код, який виконується, якщо умова істинна
} else {
    // код, який виконується, якщо умова хибна
}

Приклад:

int b = 3;
if (b > 5) {
    System.out.println("b більше за 5");
} else {
    System.out.println("b не більше за 5");
}

3. Умовний оператор if-else if-else

Іноді для перевірки декількох умов необхідно використовувати послідовність операторів if-else if-else. Це дозволяє перевіряти кілька умов і виконувати відповідні блоки коду, якщо одна з них є істинною.

if (умова1) {
    // код, який виконується, якщо умова1 істинна
} else if (умова2) {
    // код, який виконується, якщо умова2 істинна
} else {
    // код, який виконується, якщо жодна з умов не істинна
}

Приклад:

int c = 7;
if (c > 10) {
    System.out.println("c більше за 10");
} else if (c == 7) {
    System.out.println("c дорівнює 7");
} else {
    System.out.println("c менше за 7 і не дорівнює 7");
}

4. Умовний оператор switch

Оператор switch дозволяє виконувати різні блоки коду в залежності від значення певної змінної або виразу. Це зручний інструмент, коли потрібно перевірити кілька значень однієї змінної, особливо коли умови стають складними або числовими.

switch (вираз) {
    case значення1:
        // код, який виконується, якщо вираз дорівнює значенню1
        break;
    case значення2:
        // код, який виконується, якщо вираз дорівнює значенню2
        break;
    default:
        // код, який виконується, якщо жодна з умов не виконалась
}

Приклад:

int day = 3;
switch (day) {
    case 1:
        System.out.println("Понеділок");
        break;
    case 2:
        System.out.println("Вівторок");
        break;
    case 3:
        System.out.println("Середа");
        break;
    default:
        System.out.println("Некоректний день");
}

5. Порівняння умов і їх застосування

Умовні конструкції зазвичай використовуються разом з операціями порівняння, щоб визначити, чи істинні певні умови.

Операції порівняння:

  • == — рівність
  • != — нерівність
  • > — більше
  • < — менше
  • >= — більше або рівно
  • <= — менше або рівно

Приклад:

int x = 5;
int y = 10;
if (x < y) {
    System.out.println("x менше за y");
}

6. Логічні оператори в умовах

Для складніших умов можна використовувати логічні оператори, такі як:

  • && — логічне "І" (AND)
  • || — логічне "АБО" (OR)
  • ! — логічне "НЕ" (NOT)

Приклад:

int a = 5;
int b = 10;
if (a > 0 && b < 20) {
    System.out.println("Обидві умови виконуються");
}

Підсумки

Умовні конструкції є невід'ємною частиною програмування і дозволяють керувати виконанням програми в залежності від певних умов. Вони дозволяють реалізувати логіку прийняття рішень, що робить програму гнучкою та адаптивною до різних ситуацій.

Ми розглянули такі основні умовні конструкції, як if, else, switch, а також дізналися про важливі концепції, такі як операції порівняння і логічні оператори, що значно розширюють можливості умовних операторів.
