Лекція 3.1: ООП і моделювання: основи

Основи об'єктно-орієнтованого програмування та базові концепції моделювання.

Вступ

Об'єктно-орієнтоване програмування (ООП) — це парадигма програмування, яка використовує «об'єкти» для моделювання реального світу або абстракцій. Кожен об'єкт об'єднує в собі дані (властивості) і методи (функції або процедури), що працюють з цими даними. ООП дозволяє створювати гнучкі та масштабовані програми, забезпечуючи організацію коду, яка легше піддається підтримці та розвитку.

Моделювання, у свою чергу, — це процес створення спрощеної, але при цьому ефективної репрезентації системи або явища. У ООП моделювання є важливою частиною, оскільки дозволяє зобразити об'єкти та їх взаємодію в рамках програмного продукту.

В цій лекції ми розглянемо основи ООП, принципи моделювання, а також базові концепції, такі як класи, об'єкти, інкапсуляція, успадкування, поліморфізм та абстракція.

1. Основи об'єктно-орієнтованого програмування (ООП)

ООП побудоване на кількох основних концепціях, кожна з яких є важливим елементом для створення програм. Давайте розглянемо ці концепції:

1.1 Клас і об'єкт

Клас — це шаблон або модель для створення об'єктів. Клас визначає властивості і методи, якими володіє об'єкт цього класу.

Об'єкт — це екземпляр класу, що містить конкретні значення для властивостей і може виконувати методи класу.

class Car {
    // Властивості класу
    String model;
    int year;
    
    // Метод класу
    void drive() {
        System.out.println("Машина поїхала!");
    }
}

public class Main {
    public static void main(String[] args) {
        // Створення об'єкта класу Car
        Car myCar = new Car();
        myCar.model = "Toyota";
        myCar.year = 2021;
        
        System.out.println("Модель: " + myCar.model);
        System.out.println("Рік випуску: " + myCar.year);
        
        // Виклик методу об'єкта
        myCar.drive();
    }
}

1.2 Інкапсуляція

Інкапсуляція — це принцип, що передбачає захист даних об'єкта від зовнішнього доступу та зміни. Це досягається через використання модифікаторів доступу (наприклад, private, public, protected), а також через створення спеціальних методів для доступу до приватних даних (getter/setter).

class Person {
    private String name;  // Приватна змінна
    
    // Метод для доступу до приватної змінної
    public String getName() {
        return name;
    }
    
    // Метод для зміни приватної змінної
    public void setName(String name) {
        this.name = name;
    }
}

public class Main {
    public static void main(String[] args) {
        Person person = new Person();
        person.setName("Іван");
        
        System.out.println(person.getName());  // Виводить: Іван
    }
}

1.3 Успадкування

Успадкування дозволяє створювати нові класи на основі існуючих. Новий клас (підклас) може успадковувати властивості і методи іншого класу (батьківського класу), а також може мати власні властивості та методи або перевизначати методи батьківського класу.

class Animal {
    void sound() {
        System.out.println("Тварина видає звук");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Гав-гав");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal myDog = new Dog();
        myDog.sound();  // Виведе: Гав-гав
    }
}

1.4 Поліморфізм

Поліморфізм дозволяє об'єктам різних класів реагувати на однакові методи по-різному. Це досягається через успадкування та перевизначення методів. Поліморфізм дозволяє використовувати методи одного інтерфейсу або базового класу для обробки об'єктів різних типів.

class Animal {
    void makeSound() {
        System.out.println("Тварина видає звук");
    }
}

class Dog extends Animal {
    void makeSound() {
        System.out.println("Гав-гав");
    }
}

class Cat extends Animal {
    void makeSound() {
        System.out.println("Мяу");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal myDog = new Dog();
        Animal myCat = new Cat();
        
        myDog.makeSound();  // Виведе: Гав-гав
        myCat.makeSound();  // Виведе: Мяу
    }
}

1.5 Абстракція

Абстракція — це процес приховування деталей реалізації та надання лише необхідної інформації для користувача. Це досягається через абстрактні класи та інтерфейси. Абстрактні класи можуть містити абстрактні методи, які не мають реалізації, а інтерфейси визначають тільки сигнатури методів, які повинні бути реалізовані в класах.

abstract class Animal {
    abstract void makeSound();  // Абстрактний метод
}

class Dog extends Animal {
    void makeSound() {
        System.out.println("Гав-гав");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal myDog = new Dog();
        myDog.makeSound();  // Виведе: Гав-гав
    }
}

2. Моделювання в ООП

Моделювання в ООП полягає в створенні об'єктів, які відображають реальні чи абстрактні сутності. У процесі моделювання розробники визначають, які об'єкти повинні існувати в програмі, як вони взаємодіятимуть, які дані і поведінку вони повинні мати.

2.1 Моделювання через класи

Клас є основним інструментом для моделювання об'єктів у ООП. Класи визначають характеристики та поведінку об'єктів, які ми хочемо змоделювати. Наприклад, якщо ми хочемо змоделювати автомобіль, то клас Car може містити властивості (наприклад, модель, рік випуску, колір) та методи (наприклад, drive(), stop()).

2.2 Взаємодія між об'єктами

Об'єкти можуть взаємодіяти між собою, викликаючи методи один одного. Взаємодія об'єктів є важливою частиною моделювання, оскільки дозволяє створювати складні системи, де об'єкти можуть обмінюватися даними та виконувати різні функції в межах програми.

class Engine {
    void start() {
        System.out.println("Двигун запущено!");
    }
}

class Car {
    Engine engine = new Engine();  // Створення об'єкта класу Engine
    
    void drive() {
        engine.start();  // Виклик методу об'єкта Engine
        System.out.println("Машина поїхала!");
    }
}

public class Main {
    public static void main(String[] args) {
        Car myCar = new Car();
        myCar.drive();  // Виведе: Двигун запущено! Машина поїхала!
    }
}

Підсумки

Об'єктно-орієнтоване програмування є потужною парадигмою, яка дозволяє розробляти гнучкі, масштабовані та підтримувані програми. Основні принципи ООП — це інкапсуляція, успадкування, поліморфізм і абстракція — допомагають створювати ефективні моделі реального світу або абстракцій. Вони дозволяють об'єднувати дані і методи в єдину структуру, що полегшує розробку та підтримку програмних систем.
