Лекція 3.2: ООП і моделювання: класи та об'єкти

Вивчення понять класів та об'єктів у об'єктно-орієнтованому програмуванні.

Вступ

Об'єктно-орієнтоване програмування (ООП) є однією з найпоширеніших парадигм програмування, що дозволяє зручно організовувати код у вигляді об'єктів. Основними компонентами ООП є класи та об'єкти, які утворюють основну структуру для розробки програм. У цій лекції ми детально розглянемо, що таке клас і об'єкт, як їх створювати та використовувати у програмуванні.

1. Клас як шаблон для створення об'єктів

1.1 Що таке клас?

Клас — це абстракція, шаблон або конструкція, яка визначає характеристики (властивості) та поведінку (методи) об'єктів певного типу. Клас є своєрідною «рецептурою», за якою створюються об'єкти, але сам по собі не є реальним об'єктом.

Клас описує:

  • Властивості (атрибути) — це дані або характеристики об'єкта, які зберігаються в ньому.
  • Методи — це функції або процедури, що описують поведінку об'єкта, тобто операції, які об'єкт може виконувати.

1.2 Створення класу

У більшості об'єктно-орієнтованих мов програмування клас створюється за допомогою спеціальної конструкції. Ось приклад створення класу в Java:


public class Car {
    // Властивості класу
    String model;
    int year;
    
    // Метод класу
    void start() {
        System.out.println("Автомобіль завівся!");
    }
    
    void stop() {
        System.out.println("Автомобіль зупинено.");
    }
}

В цьому прикладі клас Car має дві властивості (model, year) і два методи (start(), stop()), які описують основну поведінку автомобіля.

2. Об'єкти як екземпляри класів

2.1 Що таке об'єкт?

Об'єкт — це конкретний екземпляр класу, створений за шаблоном цього класу. Кожен об'єкт має власні значення властивостей і може виконувати методи, визначені в класі. Клас лише описує, як виглядатиме об'єкт, але не є самостійним елементом.

2.2 Створення об'єкта

Щоб створити об'єкт, необхідно використати оператор створення об'єкта для класу. У Java це робиться за допомогою конструктора класу. Ось приклад створення об'єкта класу Car:


public class Main {
    public static void main(String[] args) {
        // Створення об'єкта класу Car
        Car myCar = new Car();
        
        // Призначення значень властивостей
        myCar.model = "Toyota";
        myCar.year = 2022;
        
        // Виклик методів об'єкта
        myCar.start();
        myCar.stop();
    }
}

У цьому прикладі ми створюємо об'єкт myCar типу Car, призначаємо значення його властивостям та викликаємо методи start() і stop(), що належать цьому об'єкту.

3. Різниця між класом і об'єктом

Клас — це абстракція, яка визначає структуру та поведінку об'єктів.

Об'єкт — це конкретний екземпляр класу, який існує в пам'яті і має власні значення властивостей.

Приклад: Уявіть, що клас Car — це схема для побудови автомобіля. Об'єкти класу Car будуть конкретними автомобілями з певною маркою, роком випуску тощо.

4. Інкапсуляція та доступ до властивостей і методів об'єктів

4.1 Інкапсуляція

Одним з принципів ООП є інкапсуляція, яка полягає в обмеженні доступу до внутрішніх даних об'єкта. Це дозволяє захистити властивості об'єкта від небажаних змін ззовні і дає змогу маніпулювати ними лише через спеціально визначені методи.

У Java для цього використовуються модифікатори доступу (private, public, protected). Наприклад, можна зробити властивості класу приватними, а доступ до них забезпечити через геттер та сеттер:


public class Car {
    // Приватні властивості
    private String model;
    private int year;
    
    // Публічний геттер
    public String getModel() {
        return model;
    }
    
    // Публічний сеттер
    public void setModel(String model) {
        this.model = model;
    }
    
    // Метод для виведення даних
    public void displayInfo() {
        System.out.println("Модель: " + model + ", Рік: " + year);
    }
}

Тепер, щоб змінити модель автомобіля, необхідно викликати метод setModel(), що дозволяє контролювати, як змінюється ця властивість.

4.2 Доступ до методів і властивостей

Методи та властивості класу можуть бути доступні ззовні або обмежені лише для використання всередині класу (залежно від їх модифікаторів доступу). Наприклад, якщо властивості класу Car є приватними, доступ до них можливий лише через публічні методи (геттер/сеттер).

5. Приклад моделювання з класами та об'єктами

Розглянемо приклад моделювання системи управління бібліотекою за допомогою класів і об'єктів. Ми створимо два класи: Book і Library. Клас Book описує книгу, а клас Library — бібліотеку, яка містить кілька книг.


public class Book {
    private String title;
    private String author;
    
    public Book(String title, String author) {
        this.title = title;
        this.author = author;
    }
    
    public void displayInfo() {
        System.out.println("Назва: " + title + ", Автор: " + author);
    }
}

public class Library {
    private List books;
    
    public Library() {
        books = new ArrayList<>();
    }
    
    public void addBook(Book book) {
        books.add(book);
    }
    
    public void displayBooks() {
        for (Book book : books) {
            book.displayInfo();
        }
    }
}

У цьому прикладі ми створили два об'єкти класу Book (книги) і один об'єкт класу Library (бібліотека). Бібліотека зберігає список книг, і ми можемо додавати нові книги та виводити їх інформацію.

Підсумки

Клас — це шаблон або опис об'єкта, який містить властивості та методи.

Об'єкт — це конкретний екземпляр класу, з конкретними значеннями властивостей.

Важливі концепції ООП, такі як інкапсуляція, дозволяють забезпечити захист даних і доступ до них через спеціально визначені методи.

Розуміння основ класів та об'єктів є фундаментом для глибшого освоєння ООП і розробки програм на основі цієї парадигми.

Ця лекція дає вам основу для розуміння основних понять класів та об'єктів, а також їх використання в реальних програмах.
